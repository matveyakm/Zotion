# Как внести вклад в проект

Спасибо, что хотите помочь! ❤️  
Любой вклад приветствуется: код, документация, переводы, баг-репорты, идеи, дизайн, тесты и т.д.

## Как начать

1. **Найдите задачу**  
   Посмотрите [все открытые issues](https://github.com/matveyakm/Zotion/issues).  
   Если ничего подходящего нет — создайте новый issue с описанием, что хотите сделать.

2. **Обсудите изменения**  
   Прежде чем писать много кода — напишите в issue или в обсуждении. Это сэкономит время всем.

3. **Сделайте форк и ветку**  
   - Сделайте fork репозитория  
   - Создайте ветку с понятным именем: `feature/название-фичи`, `fix/баг-123`, `docs/улучшение-readme`  
   - Внесите изменения

4. **Соблюдайте стиль кода**  
   - Следуйте текущему стилю (прогуглите «[язык] style guide», например Black для Python, Prettier для JS)  
   - Запускайте тесты / линтеры перед коммитом  
   - Делайте маленькие, атомарные коммиты с понятными сообщениями (Conventional Commits приветствуются: `feat:`, `fix:`, `docs:`, `refactor:` и т.д.)

5. **Создайте Pull Request**  
   - Заголовок PR должен быть понятным и начинаться с типа изменения (`feat: добавлен поиск`, `fix: исправлен краш при ...`)  
   - В описании укажите:  
     - Что именно изменилось  
     - Почему это важно  
     - Как тестировали  
     - Ссылки на связанные issues (`Closes #123`)  
   - Добавьте скриншоты / GIF, если меняется UI

6. **Дождитесь ревью**  
   Мы постараемся ответить в течение 1–7 дней.

## Полезные метки в issues

- `good first issue` — для новичков  
- `help wanted` — нужна помощь  
- `bug` — ошибка  
- `enhancement` — новая фича  
- `documentation` — документация

## Вопросы / связь

- Создавайте issue
- Кодекс поведения → [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md)

Спасибо за вклад! Каждый коммит делает проект лучше.
